# MorphFS

## What is MorphFS?

MorphFS is an evolving file management system built from scratch in C.

Unlike traditional tutorial projects that exist only to demonstrate concepts, MorphFS is designed as a real system that grows version by version by solving real limitations.

Every new feature is introduced because the current version needs it.

As the system evolves, it naturally introduces concepts from Data Structures, Memory Management, File Handling, System Design, and Software Engineering.

The goal is not simply to learn concepts.

The goal is to build a real product and learn concepts along the way.

---

# Vision

Most learning projects start with a concept and then build an example around it.

MorphFS takes the opposite approach.

Instead of asking:

"What project can teach Linked Lists?"

We ask:

"What limitation does the current system have?"

Then we solve that limitation using the most appropriate concept.

This creates a more natural learning experience where concepts emerge from real engineering needs.

# Current Progress

## MorphFS V1 — File Name Registry

Capabilities:

* Create file names
* Display file names

Concepts Introduced:

* Arrays

Limitation:

Files were only names and contained no real information.

---

## MorphFS V2 — Real File Awareness

Capabilities:

* Read real file metadata
* Display file information
* Work with actual files instead of dummy data

Examples:

* File Name
* File Size
* File Type
* Last Modified Date

Concepts Introduced:

* Pointers
* Dynamic Memory Allocation (malloc)
* Structures

Limitation Solved:

MorphFS can now understand real files rather than storing simple file names.

# Learning Philosophy

MorphFS is not a DSA project.

MorphFS is not a shell clone.

MorphFS is not a tutorial series.

MorphFS is a long-term engineering journey where software evolves through limitations, improvements, and real-world requirements.

The audience learns concepts because the system demands them.

Not because a syllabus says so.
